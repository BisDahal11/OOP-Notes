1. What is OOP? Describe its features.

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects", which can contain data and code: data in the form of fields (often called attributes or properties), and code in the form of procedures (often called methods). In OOP, programs are designed using objects that interact with one another. This approach models real-world entities more naturally than procedural programming.

Key features of OOP include:

  • Encapsulation: Wrapping data and code into a single unit called a class. This protects data from unauthorized access.
  • Abstraction: Hiding complex implementation details and showing only essential features to the user.
  • Inheritance: Allowing a new class to inherit properties and behavior of an existing class.
  • Polymorphism: Allowing methods to behave differently based on the object that is calling them, increasing flexibility.
  • Modularity: Dividing programs into small parts (objects), simplifying development and testing.

Languages like C++, Java, and Python use OOP for better code organization, development, and maintenance.

2. What are the advantages and disadvantages of using OOP?

Object-Oriented Programming (OOP) offers a structured and reusable way of programming. It helps manage complexity in large software systems and improves code organization through encapsulation and modularity.

Advantages:

  • Modularity: Classes can be developed and maintained independently.
  • Reusability: Inheritance allows code reuse without rewriting.
  • Maintainability: OOP promotes easier updates and bug fixing.
  • Data Security: Data hiding ensures only authorized methods can access specific data.
  • Scalability: Objects can be extended and reused for larger systems.

Disadvantages:

  • Complexity: OOP may be too complex for simple tasks or small scripts.
  • Performance: Overheads like dynamic dispatch might slow down execution compared to procedural code.
  • Learning Curve: Understanding concepts like inheritance and polymorphism takes time.

Despite some challenges, OOP is widely preferred for building maintainable and scalable applications.

3. Explain the term polymorphism and inheritance.

Polymorphism and inheritance are two fundamental concepts of Object-Oriented Programming that help make software more modular, flexible, and reusable.

Polymorphism is the ability of a single function, operator, or object to behave in different ways in different contexts. It allows the same interface to represent different underlying data types. There are two types of polymorphism: compile-time (method overloading) and run-time (method overriding). This makes the program more flexible and extensible.

For example, in C++, we can define multiple functions with the same name but different parameters (overloading). We can also override methods in derived classes to change behavior (overriding).

Inheritance allows a new class (derived or child class) to acquire the properties and behaviors of an existing class (base or parent class). It promotes code reusability and helps in organizing code logically. It also supports hierarchical classifications, where general properties are defined in base classes and specific properties in derived ones.

For example, a class Animal may have a method makeSound(), and the class Dog can inherit from Animal and override the method to produce a bark.

4. Differentiate between OOP and Structured Programming.

Structured Programming (also called Procedural Programming) and Object-Oriented Programming (OOP) are two different paradigms used to build software, each with its own design philosophy.

Structured Programming emphasizes functions and procedures. Programs are divided into procedures or functions and follow a top-down design approach. Data is passed between functions and usually has a global scope. Examples include languages like C, FORTRAN, and Pascal.

OOP, on the other hand, focuses on objects that combine both data and behavior. Programs are built from classes and objects that encapsulate data. OOP uses a bottom-up approach and emphasizes features such as encapsulation, inheritance, and polymorphism. Languages include C++, Java, and Python.

Key Differences:

  • Data Security: OOP supports data hiding via encapsulation, while structured programming usually exposes data.
  • Reusability: OOP promotes reuse through inheritance; structured programming lacks this feature.
  • Modelling: OOP models real-world entities more naturally than structured programming.

In summary, OOP is preferred for large and complex projects requiring flexibility, while structured programming is suitable for simpler, smaller applications.

5. Explain class, object, and encapsulation.

A class is a blueprint that defines the properties and behaviors of objects. It contains data members (attributes) and methods (functions) that work on the data. For example, a Car class might include attributes like color and model, and methods like drive() and brake().

An object is an instance of a class. When a class is defined, no memory is allocated until an object of that class is created. Each object has its own values for the attributes defined in the class.

Encapsulation means binding the data and the methods that modify the data together in a single unit. It also restricts direct access to some of the object's components. This is done by making attributes private and providing public methods to access them (getters and setters). It enhances data security and helps maintain a clean structure in programming.

Together, these concepts form the backbone of OOP and help in writing secure, maintainable, and reusable code.
